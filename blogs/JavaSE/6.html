    Java 内部类、代码块

    Awaken 8/31/2019 JavaSE

    # 内部类

    通过内部类可以实现类的多重继承

    在一个类A里，我们可以定义变量，也可以定义方法。思考一下，在A类里还能在定义一个B类吗？这种语法在Java里是允许的，这种情况下，B类可以被称为内部类，A类被称为外部类。

    当一个类的内部，还有一个部分需要一个完整的结构进行描述，而这个内部的完整的结构又只为外部事物提供服务，不在其他地方单独使用，那么整个内部的完整结构最好使用内部类。

    根据内部类定义的方式不同，可以讲内部类分为四种:

    • 成员内部类
    • 局部内部类
    • 静态内部类
    • 匿名内部类

    注意：对于普通的类来说，我们只能使用缺省或者public权限修饰符；而对于有些内部类，可以使用全部的四种权限修饰符

    # 成员内部类特点

    1. 定义在类里，方法外，和成员变量的位置一样。
    2. 在内部类里，可以访问外部类的所有成员变量和方法，包括private和static 需要使用外部类名.this.外部方法名
    3. 在外部类里，也可以访问内部类的所有变量和方法。 外部函数的static成员里，不允许访问成员内部类。
    4. 在内部类里，不能使用static 定义静态变量，但是可以使用static final 定义静态常量。
    5. 在内部类里如果存在和外部类同名的成员变量，在内部类里默认访问的是内部类的成员变量。
      • 在内部类里，可以使用 外部类名.this.变量名访问 外部类的成员变量。
      • 在内部类里，推荐使用 外部类名.静态变量名 访问 *外部类的静态变量。
    6. 定义个普通类的时候，只可以使用 public 和缺省 权限修饰符 ，以及final abstract 修饰符。
      • 但是作为一个成员内部类来说，可以使用四种权限修饰符。 public/protected/缺省/private/fianl/abstract
    7. 怎样创建一个成员内部类对象?
      • 在外部类里创建，并返回这个内部类对象
      • 直接在外部无关类创建一个内部类对象，如果内部类被权限修饰符控制了，外部可能不能直接创建对象。
    8. 成员内部类在编译时也会生成一个 .class 文件 ,但是这个文件的名字是 外部类名$内部类名.class
    public class MemberInnerDemo {

        Outer1 outer1 = new Outer1();

    # 局部内部类特点

    1. 定义在方法里的内部类，称为局部类。

    2. 局部内部类可以访问外部类的成员变量。

      如果局部内部类的成员变量和外部类的成员变量同名，在局部内部类里可以使用 外部类名.this.变量名访问外部类的成员变量。

    3. 可以访问外部方法的局部变量。

    条件：

    • 但是如果局部内类的成员变量和外部方法的局部变量重名，会访问自己的内部类的成员变量，无法在访问外部类的局部变量。
    • 外部方法的局部变量，需要被final修饰，在内部类里不允许修改。在JDK1.8 以后，如果一个局部变量被局部内部类访问了，可以认为这个变量会被final修饰。

    局部成员变量不能使用任何的权限修饰符，只能使用缺省修饰符。 不能在局部内类使用static 定义静态成员变量，但是可以使用static final 定义静态常量！ 如果局部内部类所在的方法是静态，不能访问非静态的属性。 局部内部类也可以被编译成为 .class 文件，文件名是 外部类名$1局部内部类名 。

    # 匿名内部类（最常用的内部类）

    它其实就是一个局部内部类，只不过这个局部内部没有名字

    接口 和 抽象类 不能直接创建对象，除非在创建对象的时候实现抽象方法

    匿名内部类就是用来快速的创建一个 接口或者抽象类 对象

    public class AnonymityInnerClass {
    public static void main(String[] args) {
        // 不能直接创建一个接口对象
//        AInterface aInterface = new AInterface();

        AInterface aInterface = new AInterface() {
            @Override
            public void a() {
                System.out.println("hello");
            }
        };

        // aInterface 这个对象是什么类型的对象?
        // 不是 AInterface 类型，而是一个匿名的内部类，这个类没有名字，但是它实现了 AInterface 接口。
        System.out.println(aInterface.getClass());
        System.out.println(aInterface instanceof AInterface);  // true

        // 创建一个接口对象:
        // 1. 创建一个类，让这个类实现 接口，实现接口里的抽象方法。创建这个子类对象
        // 2. 使用匿名内部类直接创建一个 接口 的实现类对象。 匿名对象匿名内部类
        // Person p = new Person();
        // p.eat();
        // p.sleep();
        // new Person().eat();
        // new Person().sleep();
        Person p1 = new Person("安妮", 18, 1.7);
        Person p2 = new Person("崔崔", 16, 1.68);
        Person p3 = new Person("芳芳", 17, 1.66);
        Person p4 = new Person("朦朦", 19, 1.72);
        Person p5 = new Person("弛弛", 20, 1.75);
        Person[] people = new Person[]{p1, p2, p3, p4, p5};
        System.out.println(Arrays.toString(people));

        // 如何对一个对象数组进行排序? 关键点在于指定对象按照哪种方式来排序
        // 1. 让对象实现 Comparable 接口，变成可比较的对象,并重写 compareTo 方法，在这个方法里指定排序的规则
//        Arrays.sort(people);
        // 2. 对象不实现 Comparable 接口，在比较的时候传入一个比较器。
        /*Arrays.sort(people, new MyComparator());
        Arrays.sort(people, new MyComparator2());*/

        /*Arrays.sort(people, new Comparator<Person>() {
            @Override
            public int compare(Person o1, Person o2) {
                return o1.getAge() - o2.getAge();
            }
        });*/
        // 匿名对象，而且这个匿名对象的类型是匿名内部类
        Arrays.sort(people, new Comparator<Person>() {
            @Override
            public int compare(Person o1, Person o2) {
//                return (int)(o1.getHeight() - o2.getHeight());
                if (o1.getHeight() > o2.getHeight()) return 1;
                else if (o1.getHeight() == o2.getHeight()) return 0;
                return -1;
            }
        });
        System.out.println(Arrays.toString(people));
    }
}

    # 静态内部类

    特点:

    1. 定义的位置和成员内部类的位置一样，只是在成员内部类的前面加上static
    2. 静态内部类可以使用static关键字定义静态变量了。
    3. 静态内部类不能访问外部类的非静态变量。 静态内部类不持有外部对象的引用，可能外部类都没有实例化。
    4. 可以不创建外部类对象，直接创建内部类对象。

    # 代码块

    Java中包括四种代码块:

    1. 普通代码块：就是类里方法体代码。
    2. 构造代码块：类里直接使用 {} 编写的代码。
    3. 静态代码块：在构造代码块前添加static关键字。
    4. 同步代码块：使用synchronize关键字包裹起来的代码块，用于多线程。

    普通代码块就是方法体的代码，这里就不再赘述。下面我们来看一下构造代码块和静态代码块

    # 静态代码块

    特点:

    1. 使用static关键字修饰，写在类里，方法外，用来对类进行初始化。
    2. 一个类里可以有多个静态代码块，但是通常情况下只会定义一个。
    3. 随着类加载而执行，只执行一次，只要类加载就会执行，所以执行的优先级很高。
    4. 一般情况下,如果有些代码必须在项目启动的时候就执行的时候,就需要使用静态代码块。

    # 类的初始化

    类被加载内存后，会在方法区创建一个Class对象（后面反射章节详细学习）来存储该类的所有信息。此时会为类的静态变量分配内存，然后为类变量进行初始化。在加载类的过程中，JVM会调用方法，专门来对类变量或者静态代码块进行初始化。

    public class Test{
    public static void main(String[] args){
        Father.test();
    }
}
class Father{
    private static int a = getNumber();
    static{
        System.out.println("Father(1)");
    }
    private static int b = getNumber();
    static{
        System.out.println("Father(2)");
    }

    public static int getNumber(){
        System.out.println("getNumber()");
        return 1;
    }

    public static void test(){
        System.out.println("Father:test()");
    }
}

运行结果：
getNumber()
Father(1)
getNumber()
Father(2)
Father:test()

    package com.atguigu.java;

public class Demo {
    public static void main(String[] args) {
        System.out.println("我是main方法");

        Demo d1 = new Demo();

        return name;

    特点：

    # 实例对象的初始化

    除了方法初始化类以外，在创建实例对象时，还会自动调用方法对实例对象进行初始化。

    使用一个类来实例化对象的几种方式:

    1. 最常见的方式，使用关键字new来创建一个实例对象。
    2. 使用反射，调用Class或者java.lang.reflect.Constructor对象的newInstance()方法。
    3. 调用对象的clone()方法。
    4. 调用java.io.ObjectInputStream类的getOjbect()方法序列化。
    public class Test{
    public static void main(String[] args){
        Father f1 = new Father();
        Father f2 = new Father();
    }
}
class Father{
    private int a = getNumber();
    private String info;
    {
        System.out.println("Father(1)");
    }
    Father(){
        System.out.println("Father()无参构造");
    }
    Father(String info){
        this.info = info;
        System.out.println("Father(info)有参构造");
    }
    private int b = getNumber();
    {
        System.out.println("Father(2)");
    }

    public int getNumber(){
        System.out.println("Father:getNumber()");
        return 1;
    }
}

    # 类和对象的加载过程

    # 对象的初始化过程:

    1. 给成员变量设置默认值。 Son里 int x = 0;
    2. 调用构造方法，初始化父类。 2.1 父类给成员变量设置默认值。 Father里int x = 0; 2.2 调用构造方法，初始化父类。 2.2.1（如果还有父类就继续套娃） 2.3 给成员变量赋值以及执行构造代码块。 Father里的int x=10; 2.4 调用构造方法里的其他代码。 Father里的int x = 20;
    3. 给成员变量赋值以及执行构造代码块。 Son里的 int x = 30;
    4. 调用构造方法里的其他代码。

    # 类的初始化过程：

    一个类什么时候会被加载(反射) a. 创建对象的时候会加载类 b. main 方法所在的类也会被加载 c. 静态变量被使用的时候.

    1. 给静态成员变量设置默认值。
    2. 初始化父类。
    3. 执行构造代码以及给静态成员变量赋值(按照代码的书写顺序来执行)。
