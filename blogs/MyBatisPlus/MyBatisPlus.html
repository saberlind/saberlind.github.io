    2、带条件的查询

    QueryWrapper/LambdaQueryWrapper的使用

    //根据其他条件查询
@Test
public void testQueryByCondition(){
    //查询姓名中包含a的 并且年龄>20岁的用户 按照年龄降序排列
    // queryWrapper用来封装查询条件，并提供将跳转转为动态sql语句的方法
    QueryWrapper<User> queryWrapper = new QueryWrapper<>();
    //        queryWrapper.likeLeft("name" , "a") 在a的左边拼接%
    //        queryWrapper.likeRight()
    //name代表数据库中表的字段名
    queryWrapper.like("name" , "a");//在a的左右两边拼接%
    queryWrapper.gt("age" , 20);

    queryWrapper.orderByDesc("age");
    List<User> users = userMapper.selectList(queryWrapper);

    users.forEach(user->{
        System.out.println("user = " + user);
    });
}

@Test
public void testQueryByCondition2(){
    //查询年龄>20岁的用户 或者姓名中包含e的 按照年龄降序排列   只查询年龄姓名和id
    QueryWrapper<User> queryWrapper = new QueryWrapper<>();
    queryWrapper.gt("age" , 20);
    queryWrapper.or().like("name" , "e");
    queryWrapper.orderByDesc("age");
    queryWrapper.select("id" , "age" , "name");
    //只查询前三条记录
    queryWrapper.last("limit 0 , 3");
    List<User> users = userMapper.selectList(queryWrapper);
    users.forEach(System.out::println);
}


@Test
public void testQueryByCondition3(){
    //多表联查 一般使用QueryWrapper： 需要指定某个列从那张表中获取，需要自定义列名时使用
    //单表操作 一般使用LambdaQueryWrapper
    // mp支持lambda表达式解析，可以使用类的属性解析 类属性在表中对应的字段名
    //查询年龄>20岁的用户 或者姓名中包含e的 按照年龄降序排列   只查询年龄姓名和id
    LambdaQueryWrapper<User> queryWrapper = new LambdaQueryWrapper();
    queryWrapper.gt(User::getAge , 20);
    queryWrapper.or().like(User::getName , "e");
    queryWrapper.orderByDesc(User::getAge);
    queryWrapper.select(User::getAge , User::getId , User::getName);
    queryWrapper.last("limit 3");
    userMapper.selectList(queryWrapper);

}

    # 5、mp-新增

    数据库访问压力大，解决方案：

    1、分布式： 分库

    ​ 将一个库中的表拆分到不同的库中存储，每个微服务只操作自己的数据库

    2、水平拆分+垂直拆分：

    ​ 垂直拆分： 将一张表不常用的大字段拆分到其它表中存储，拆分出去的表也可以使用以前表的主键值当做自己的主键

    ​ 水平拆分： 将一张表的数据 存到不同库的表中，表结构都一样

    ​ 如何保证id唯一

    ​ 全局序列id(有序、过滤性强、全局唯一)

    @Data
@Accessors(chain = true) //链式调用
@TableName(value = "user")  //表示表名：如果表名和类名形成了映射 可以省略注解，如果不一样需要绑定表名和类名
public class User {
    //type = IdType.ASSIGN_ID 默认：使用雪花算法生成id
    //ASSIGN_UUID: 使用UUID生成id
    //AUTO: 数据库自增长id
    @TableId(value = "id",type = IdType.ASSIGN_ID)  //表示当前字段为数据库的主键
    private String id;
    @TableField(value = "name") //表示当前字段为数据库的一个普通列
    private String name;
    private String email;
    private Integer age;
}

public class GlobalMetaObjectHandler implements MetaObjectHandler {
    //新增时 自动填充回调方法
    @Override
    public void insertFill(MetaObject metaObject) {
        metaObject.setValue("gmtCreate" , new Date());
        metaObject.setValue("gmtModified" , new Date());

    //更新时 自动填充回调方法
    @Override
    public void updateFill(MetaObject metaObject) {
        metaObject.setValue("gmtModified" , new Date());

    # 9、mp-删除+逻辑删除

    以后重要的数据删除时都使用逻辑删除

    1、在表中添加逻辑删除字段

    ALTER TABLE USER add COLUMN is_deleted CHAR(1) DEFAULT '0';
    1

    2、在javabean中添加逻辑删除字段 并 添加@TableLogic注解

    @TableField(value = "is_deleted")
@TableLogic //表示逻辑删除字段：以后删除时会自动更新该字段的值，查询时会使用该字段作为条件
private Boolean deleted;
    1
    2
    3

    3、以后查询时会自动添加逻辑删除字段作为条件

    SELECT *,is_deleted AS deleted FROM user WHERE is_deleted=0
    1

    删除时更新is_deleted字段的值

    update user set is_deleted = 1 where id = 1 and is_deleted = 0
